# Быстрый старт

Гайд для запуска проекта без глубокого погружения в архитектуру.
Подходит для Windows, macOS и Linux.

## 1. Что нужно установить

- Git
- Docker Desktop (Windows/macOS) или Docker Engine + Docker Compose plugin (Linux)

Дополнительно:
- Windows: включите WSL2/VirtualMachinePlatform (обычно нужна перезагрузка).

## 2. Клонирование и подготовка

### macOS / Linux (bash/zsh)

```bash
git clone https://github.com/MaksimSamarin/ai-feedback-analysis.git
cd ai-feedback-analysis
cp .env.example .env
```

### Windows PowerShell

```powershell
git clone https://github.com/MaksimSamarin/ai-feedback-analysis.git
cd ai-feedback-analysis
Copy-Item .env.example .env
```

## 3. Рекомендуемые настройки для 16 CPU / 32 GB RAM

В `.env` выставьте:

```bash
GLOBAL_LLM_PARALLELISM=12
OPENAI_PROVIDER_LABEL=OpenAI
# OLLAMA_BASE_URL=http://host:11434
# OLLAMA_MODELS=llama3.1:latest
```

Почему так:
- это рабочий старт с запасом под рост нагрузки;
- если не хватает скорости, увеличивайте сначала `worker` в `--scale`.

## 4. Лимиты контейнеров (docker-compose)

Рекомендуемые лимиты:
- `postgres`: `cpus: 4`, `mem_limit: 4g`
- `redis`: `cpus: 1`, `mem_limit: 1g`
- `backend` (на контейнер): `cpus: 1`, `mem_limit: 1g`
- `worker` (на контейнер): `cpus: 1`, `mem_limit: 2g`
- `inspect-worker` (на контейнер): `cpus: 1`, `mem_limit: 1500m`
- `frontend`: `cpus: 1`, `mem_limit: 512m`

Если сервер слабее, уменьшайте в первую очередь число `worker`.

## 5. Запуск

### Универсально (любой OS)

```bash
docker compose --env-file .env --env-file .env.versions up -d --build --scale backend=2 --scale worker=4 --scale inspect-worker=2 --scale frontend=1
```

Проверка:

```bash
docker compose ps
docker stats
```

## 6. Куда заходить

- UI: `http://localhost:8080`
- Health API: `http://localhost:8080/api/health`

Логотип:
- По умолчанию используется встроенный логотип.
- Для своего логотипа положите файл в `branding/logo.svg` или `branding/logo.png` (они не коммитятся в git) и перезапустите frontend:
- `docker compose up -d --build frontend`
- Приоритет файлов: `logo.svg` → `logo.png` → `logo-default.png` → встроенный `logo-default.svg`.

## 7. Первый анализ — по шагам

1. Откройте UI, зарегистрируйтесь и войдите.
2. **Блок «1. Данные и модель»** — перетащите `.xlsx` в зону загрузки или выберите через диалог.
   - Укажите лист, если их несколько.
   - **«Колонки для анализа»** — содержимое которых уйдёт в модель.
   - **«Колонки из исходника в итоговый отчёт»** — поля, которые попадут в итоговый xlsx без отправки в модель (ID, дата, автор).
   - **Группировка** (опционально): можно сгруппировать строки по значению выбранной колонки — например, по смене или магазину. Модель получит всю группу целиком и вернёт один ответ на группу.
   - Если нужно поменять провайдера, модель или токен — раскройте «Продвинутые настройки LLM». По умолчанию уже подставлен рабочий набор.
3. **Блок «2. Шаблон»** — напишите промпт и настройте поля ответа.
   - Быстрый путь: в выпадающем списке «Пресет» выберите готовый пример. Сейчас доступны два:
     - **Анализ мошенничества (по одному отзыву)** — построчная классификация: категория, тип нарушения, уверенность, описание.
     - **Анализ мошенничества по магазинам (группировка)** — агрегат по группе: число нарушений, доля, схемы, уровень риска, описание.
   - Поля ответа можно называть по-русски, указывать тип (текст, число, дата, список-выбор, массив, да/нет), диапазоны и ограничения длины.
   - Подставлять данные строки в промпт вручную не нужно — сервис сам передаёт содержимое каждой строки модели.
   - Если в шаблоне что-то непонятно — нажмите чип **«Как это работает»** рядом с заголовком блока: там пошаговая инструкция.
4. **Блок «3. Обработка»** — нажмите **«Запустить новый анализ»**. Во время работы доступны «Пауза», «Продолжить», «Остановить» и «Перезапустить» (для завершённых с ошибкой).
5. **Блок «4. Результаты»** — превью первых обработанных строк появляется уже в процессе. По ним удобно сразу проверить качество промпта, не дожидаясь конца. Полные данные — в xlsx-выгрузке (кнопка «Скачать»). Промежуточную выгрузку можно получить в любой момент.

Демо-файлы в `examples/`:
- `Мошенничество_1000_отзывов.xlsx` — 1 000 отзывов для быстрого теста.
- `Мошенничество_100000_отзывов.xlsx` — 100 000 отзывов для нагрузочного прогона.
Колонки: `id`, `магазин`, `смена`, `отзыв`, `оценка`, `дата`. Группировка по `смена` — рабочий сценарий (группы ~45 отзывов), группировка по `магазин` — негативный сценарий (группы ~20 000 отзывов не влезают в контекст модели, сервис вернёт понятную ошибку).

## 8. Если тормозит

1. Проверьте `docker stats`.
2. Уменьшите:
   - `worker` в `--scale`,
   - `inspect-worker` в `--scale`,
   - `GLOBAL_LLM_PARALLELISM`.
3. Проверьте, что в «Колонки для анализа» выбраны поля с полезным содержимым, а не только ID и служебные атрибуты.
4. Если после загрузки файла долго висит `parsing`:
   - это обычно означает, что `inspect-worker` заняты и файл ждёт inspect-очередь;
   - увеличьте `--scale inspect-worker=...` или дождитесь освобождения воркера.

## 9. Остановка

```bash
docker compose down
```
