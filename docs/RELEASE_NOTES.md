# Заметки о релизах

Краткая сводка по версиям. Сначала главные изменения, потом мелкие улучшения, в конце — техническая часть.

---

## 2.0.0

Первый публичный релиз. Приложение перестало падать на больших файлах, конструктор полей стал свободным, групповой режим стал рабочим инструментом, UI перестроен вокруг понятного пути «файл → шаблон → запуск → результат», добавлены заметки о релизе и позиция в очереди прямо в UI.

### Главное

- **Большие отчёты доходят до конца.** Отчёты на 200 000–700 000 строк больше не падают по нехватке памяти при финализации: запись xlsx и raw JSON идёт потоком, RAM не зависит от размера отчёта.
- **Свобода в конструкторе полей.** Убраны обязательные системные поля (`summary`, `category`, `confidence`). Пользователь сам определяет имена, типы и ограничения. Поля можно называть по-русски.
- **Снят лимит длины промпта и размера группы.** 100 000 строк в группе по умолчанию, длина промпта не навязывается. Если не влезает в контекст модели — честная ошибка с именем модели, а не молчаливая обрезка.
- **Перезапуск упавших отчётов без повторного расхода токенов.** Кнопка «Перезапустить» на отчётах `error`/`canceled`: сбрасывает в `pending` только упавшие и скипнутые строки, уже готовые строки не переспрашиваются, кэш принудительно включён.
- **Умный retry: модель узнаёт о своей ошибке.** При невалидном ответе в повторный запрос добавляется блок с причиной от валидатора и прошлый ответ модели — закрывает 60–80% валидационных ошибок на второй попытке.
- **Live-превью во время обработки.** Первые обработанные строки появляются в блоке «4. Результаты» сразу и обновляются каждые 5 секунд — можно проверить качество промпта и не ждать 30 минут до конца.
- **Воркер переживает рестарт Postgres.** Разрыв соединения с БД обрабатывается: пул пересоздаётся, задача возвращается в очередь, обработка продолжается автоматически. Никаких ручных `docker compose restart` больше не нужно.
- **Промежуточная выгрузка прямо в процессе.** Кнопка «Скачать промежуточный…» собирает xlsx по текущему состоянию БД — для больших отчётов можно показать коллеге первый срез, не тратя все токены.

### Групповой режим

- Группировка по любой колонке исходного файла (смена, оператор, магазин, день и т.д.).
- В xlsx-выгрузке — одна строка на группу; первая колонка называется `group_number` и содержит порядковый номер (1..N).
- В превью и в xlsx для группы остаётся только колонка группировки + агрегат LLM. Поля одной случайной записи (например, «отзыв», «оценка», «дата») не мешаются.
- Счётчики в «4. Результаты» честно считают группы в карточке «Всего групп X/Y», а «Успешно/С ошибками» — по строкам (совпадает с donut-диаграммой).
- Сервис показывает количество групп сразу при выборе колонки группировки — до запуска понятно, сколько работы.

### UI

- **Блоки перегруппированы и переименованы:** «1. Данные и модель» → «2. Шаблон» → «3. Обработка» → «4. Результаты».
- **Форма запуска сведена в один блок.** Минимум на экране: dropzone + 4 селекта (Лист / Колонки для анализа / Колонки в итоговый отчёт / Группировка). Провайдер, модель, токен, лимиты, параллелизм, температура — под раскрывающейся панелью «Продвинутые настройки LLM», закрытой по умолчанию.
- **Два готовых пресета для тестирования сервиса** под задачу поиска мошенничества в отзывах: построчный и групповой (по магазинам/сменам). Все поля ответа — на русском. Ориентир: крупные instruct-модели на групповом пресете дают ~95% правильных оценок риска; маленькие модели на сложных правилах с несколькими условиями сильно промахиваются.
- **Таблица «Мои отчёты»:** цветные статусы, ETA словами («~44 сек», «~1 ч 15 мин»), прогресс для групповых в объектах. Частые действия (Пауза/Продолжить/Перезапустить/Скачать) — цветными иконками, редкие — в меню `⋮`.
- **Подсказка «Как это работает»** у шаблона — цветной чип рядом с заголовком блока. Пошаговая инструкция разворачивается по клику.
- **Drag-scroll на превью:** зажал ЛКМ и тянешь — таблица скроллится горизонтально, курсор меняется на «кулак». Tooltip с полным текстом ячейки и обычный клик не ломаются.
- **Диалог скачивания** с выбором формата (XLSX / raw JSON / исходный файл) и своим именем файла (дефолт `report_<id>_YYYY-MM-DD`). Слэши, кавычки и управляющие символы в имени автоматически убираются — имя всегда валидное для файловой системы, расширение приклеивается по формату.
- **Блок «Загрузка файла»** переделан: кликабельная ссылка «загрузите вручную» (в фирменном красном цвете) + явно указанный формат «XLSX». После загрузки dropzone превращается в карточку с именем файла и ссылкой «загрузить другой».
- **Сервис запоминает последний выбранный провайдер и модель** в браузере — при повторном заходе не нужно снова выбирать свою рабочую модель.
- **Редизайн страницы отчёта:** три карточки (всего / успешно / с ошибками), круговая диаграмма «успех vs ошибка», таблица превью с динамическими колонками из вашей схемы.

### Качество работы с моделями

- **Кэш автоматически чистится от устаревших ответов.** Если вы поменяли поля ответа или промпт, старый ответ из кэша больше не подходит под новую схему. Раньше сервис всё равно возвращал его, и строка падала в ошибку на каждом запуске. Теперь сервис сразу видит, что ответ из кэша не подходит, удаляет его и делает свежий запрос в модель.
- **Retry с обратной связью всегда идёт в модель, минуя кэш.** Иначе повторная попытка получала бы из кэша тот же неправильный ответ, на который мы как раз даём модели обратную связь.
- **Если промпт не влез в контекст модели — сразу понятная ошибка, без повторных попыток.** Это не временный сбой: повтор с тем же длинным промптом приведёт к тому же результату. Пользователь видит сообщение с именем модели и подсказкой (уменьшить размер группы или взять модель с большим контекстом).
- **Числа в Excel — как числа.** `int`/`float`/`bool` из JSON сохраняются типом `Number`/`Boolean`, с ними работают формулы и фильтры.

### Демо-файлы

- `examples/Мошенничество_1000_отзывов.xlsx` — 1 000 отзывов, быстрый тест (~67 KB).
- `examples/Мошенничество_100000_отзывов.xlsx` — 100 000 отзывов, нагрузочный прогон (~6 MB).
- Колонки: `id`, `магазин`, `смена`, `отзыв`, `оценка`, `дата`. Группировка по `смена` — рабочий сценарий (группы ~45 отзывов). Группировка по `магазин` — негативный сценарий (группы ~20 000 не влезут в контекст → понятная ошибка).
- Тексты отзывов — уникальные шаблоны × случайные подстановки (имена, суммы, товары), фоновый процент мошенничества ~6%.

### Технические детали (коротко)

- **Стриминговый экспорт.** `iter_report_rows` (keyset-пагинация, `WHERE row_number > :last ORDER BY row_number ASC LIMIT :batch` — O(n), без OFFSET-деградации) + `Workbook(write_only=True)` + потоковый JSON. На 200k строк дельта RSS при финализации ~0.5 МБ.
- **Массовое обновление строк группы.** `bulk_update_report_rows_same_result` через `row_number = ANY(?)` батчем 5 000 — один UPDATE вместо N.
- **Быстрая вставка placeholder-строк.** psycopg3 `COPY FROM STDIN` на чистой таблице с fallback `INSERT ... ON CONFLICT DO NOTHING`.
- **Единый Redis-клиент** на весь backend вместо независимых пулов по модулям.
- **Публикация progress-событий** — через shared-клиент и `asyncio.to_thread`, без TCP-handshake на каждое событие.
- **Partial-индекс** `idx_report_rows_group_key (report_id, group_key, status) WHERE group_key IS NOT NULL` — ускоряет групповые запросы, не раздувает негрупповые.
- **Явный rollback** в контекст-менеджере `get_conn()` с защитой от падения самого rollback.
- **SQL-адаптер** `?` → `%s` через regex — не ломает строковые литералы, JSON и комментарии.
- **Отложенная очистка** `JobManager.jobs` через `JOB_CLEANUP_DELAY_SEC` (300 сек) — фикс медленной утечки памяти.
- **Единый источник версии** — `backend/app/__init__.py::__version__`, используется в OpenAPI и в метаданных raw JSON.
- **Удалён мёртвый код:** endpoint `/api/schema/suggest` + `_suggest_schema_and_prompt` (~170 строк), валидация `{row_json}` как обязательного плейсхолдера, старые пресеты sentiment/support_triage/classification/extraction, функция `enforce_reports_limit`.
- **ENV-переменные удалены:** `MAX_REVIEW_CHARS`, `LLM_MAX_PROMPT_CHARS`, `LLM_SKIP_OVERSIZED_REQUESTS` (если были в `.env` — можно убрать).
- **Тесты.** 225+ зелёных юнит-тестов в 28 файлах, прогон ~12 сек. Каждый баг в релизе закрыт регрессионным тестом.

### Что впереди

- Советник по промптам в UI — встроенный AI-помощник, который знает как устроено приложение и подсказывает: как сформулировать промпт, какие поля ответа выбрать, когда брать группировку и какая модель подойдёт под задачу.
- Встроенный BI: pivot-таблицы и графики по результатам отчёта прямо на странице отчёта, без выгрузки в сторонние инструменты.
- Иерархическая группировка для больших групп: если группа не влезает в контекст модели, сервис разобьёт её на под-группы, получит промежуточные агрегаты и сделает финальный запрос поверх них.
- Переработка админки: dashboard, статистика по токенам и ошибкам, чтение логов сервисов прямо в UI.
- Поддержка CSV-файлов для загрузки (сейчас только XLSX).
- Опциональное уведомление о завершении отчёта (почта или мессенджер).
